– How is an image converted into numbers?
    1. Converted to RGB,
    2. Resized to 64 x 64,
    3. Converted to a NumPy array,
    4. Scaled to the range [0, 1],
    5. Flattened into a 1D feature vector.

– What does the model actually “see”?
    It sees raw pixel intensities arranged as a flat numeric vector.
    No objects, no shapes, no semantic meaning — just numeric patterns.

– Why can the model classify random or unknown images as one of the known classes?
    All of them are standard classifiers that always try to assign one of the known labels from training. They do not have an “unknown” or “reject” option.
    So if you feed them a random image, they will still pick the closest match in feature space.

– Why can this behavior be dangerous in real AI systems?
    This is dangerous because a system may sound confident even when it has never learned the input it’s seeing. In practice, that can lead to:
        false positives,
        unsafe decisions,
        overtrust by users,
        poor behavior on out-of-distribution data,
        failure in real-world edge cases.

– What would need to be improved before using such a system operationally?
    To use such a system operationally, we would need to:
        1. Implement an “unknown” class or confidence threshold to reject out-of-distribution inputs.
        2. Train on a more diverse dataset that includes edge cases and random noise.
        3. Use techniques like anomaly detection to identify when the model is seeing something it hasn’t learned.
        4. Continuously monitor and update the model based on real-world performance.
